Introduction

IDLab is a comprehensive desktop application designed to simplify the creation of professional ID cards for schools Primary and Secondary. Developed by Emmanuel Ngwenyama, IDLab addresses the challenges faced by Zambian schools, particularly when generating examination ID cards for the candidates.

Traditional ID card creation often requires complex design software like Photoshop or CorelDRAW, which can be expensive and difficult to learn. IDLab provides an all-in-one solution with an intuitive interface that manages candidates data, photos, and card design in one place, making the entire process faster and more efficient.

IDLab works completely offline, allowing schools in areas with limited internet connectivity to create professional-quality ID cards for their candidates and staff without requiring cloud services but the user is required to activate their license.

Profile Management

Installation Guide

System Requirements

  • Windows 7 or higher (Windows 10/11 recommended)
  • 4GB RAM minimum (8GB recommended)
  • 100MB free disk space
  • 1024x768 screen resolution or higher

Installation Steps

  1. Download the Installer:IDLab installer Click to Download IDLab and save the file to a location you can easily access
  2. Run the Installer: Locate the downloaded file (usually named "IDLab_Installer.exe"), double-click to run the installer. If prompted by Windows security, click "Yes" to allow the installation
  3. Follow Instructions: Follow the on-screen instructions to complete the installation
  4. First Launch: Launch IDLab from the desktop shortcut or Start menu. The application will open with a splash screen and prompt you to set up your profile
  5. Activate License: If you have purchased a license, go to Help → Activate License and enter your license key to unlock full features

Setting Up Your Profile

Create a New Profile

  1. When first launching IDLab, the Profile Setup window appears
  2. Select your user type: School or Business
  3. Enter your school/business name, district, and address
  4. Click "Save Profile"

Switch Between Profiles

  1. Go to Settings → Switch Profile
  2. Select a profile from the list
  3. Click "Switch Profile"

Managing Candidates Data

Importing Candidates Data

  1. Go to the "Pupil Data Entry" tab
  2. Click the "Import Pupil List" button (or use Pupils → Import From Provisional Registers)
  3. Select your CSV file (must include columns: Name, School, Grade/Form, Exam Number, Sex, D O B)
  4. Review imported data in the table

Manually Adding Candidates

  1. In the "Pupil Data Entry" tab, click "Add Pupil"
  2. Fill in the candidate details in the form
  3. Click "Save" to add the candidate to your database

Editing Candidate Records

  1. Double-click on any cell in the data table to edit
  2. Make your changes and press Enter to save
  3. Use Ctrl+Z to undo changes if needed

Searching and Filtering

  • Use the search box to find specific students
  • Use the grade/form dropdown to filter by level
  • Click "Clear" to reset filters

Exporting candidates Data (CSV)

  1. Go to Pupils → Export Pupil List
  2. Choose a location to save your CSV file
  3. Transfer the file to your Android device using a USB cable or bluetooth
  4. Open IDLab on your Android device Import the .Csv by tapping the top right corner of the file icon on the data screen
  5. Follow the prompts to complete the import process
  6. Once imported, you can tap a candidate to open the camera and Capture their photo. continue until all candidates have been photographed.
  7. After capturing photos, you can export the photos back to your android device by going to the Gallery screen and tapping the Download icon at the top right corner of the screen.
  8. Transfer the photos to your computer using a USB cable or bluetooth for further processing in the desktop application. the photos can be found in pictures folder on your android device.

Managing Photos

Setting Up the Photos Folder

  1. After transferring candidate photos to your computer, create a new folder for them
  2. In the "Generate IDs" tab, click "Photos Folder"
  3. Select a folder containing your candidate photos
  4. Photos should ideally be named with exam numbers for automatic matching
  5. Click "OK" to confirm your selection
  6. IDLab will scan the folder and attempt to match photos to candidates based on exam numbers

Using the Renaming Wizard

  1. Go to the "ID Photo Tools" tab
  2. Click on the "Renaming Wizard" sub-tab
  3. Browse unmatched photos in the gallery
  4. Find and select the correct student from the list
  5. Click "Assign Photo to Pupil"

Checking Photo Status

  1. Go to the "Management" sub-tab
  2. Review which students have photos and which don't
  3. Use "Export Unmatched Lists" to create lists of students missing photos

Generating ID Cards

Basic Card Setup

  1. Go to the "Generate IDs" tab
  2. Set up your photo folder and output folder
  3. Add your school logo and coat of arms (optional)
  4. Choose between custom template or built-in template

Customizing Card Design

  1. Enter your school motto and year
  2. Choose your border color, details font color, and header font color
  3. Click "Refresh Preview" to see your changes

Setting Back Information

  1. Enter the text for the back of the ID cards in the "Back Info" field
  2. This text will appear on the reverse side of the cards

Generating Cards

  1. Select a category from the dropdown (Grade 7, Grade 9, etc., or "All")
  2. Click "Export to PDF" to create a multi-card PDF
  3. Or use "Export Images Only" from the Pupils menu to create individual image files

Back Side PDF

  1. Click "Export Back Info to PDF" to create a separate PDF for the back of the cards

Android App Installation

  • 50MB free storage space
  • Camera access for photo capture
  • Storage access for photo management

Installation Steps

  1. Download the IDLab APK file from our official website
  2. Enable "Install from Unknown Sources" in your Android settings

  4. Locate and tap the downloaded APK file

  2. Open sync settings in the mobile app menu
  3. Select "Connect to Desktop" option
  4. Enter the sync code displayed on your desktop app
  5. Choose sync direction (mobile to desktop, desktop to mobile, or two-way)
  6. Initiate synchronization process

Cloud Backup Options

  • Automatic backup to Google Drive
  • Real-time synchronization across devices
  • Easy data restoration on new devices
  • Encrypted data transmission and storage

Troubleshooting Sync Issues

  • Check network connectivity on both devices
  • Restart both apps if sync fails
  • Ensure both apps are updated to latest versions
  • Check firewall settings on desktop computer

Privacy: All data synchronization is encrypted and occurs directly between your devices. No personal information is stored on external servers.

Troubleshooting

Common Issues

Issue: Photos not appearing on ID cards
Solution: Ensure photos are in the correct folder selected as "Photos Folder". Check that photos are named exactly with the student's exam number. Use the Renaming Wizard to manually match photos with students. Verify photos are in JPG, JPEG, or PNG format.
Issue: CSV import fails
Solution: Ensure your CSV file has the required columns: Name, School, Grade/Form, Exam Number, Sex, D O B. Check for special characters or formatting issues in your CSV. Try opening and resaving your CSV in Excel or a text editor before importing.
Issue: PDF export is slow or crashes
Solution: Reduce the number of cards by filtering to a specific grade. Ensure you have enough disk space for the output. Try exporting in smaller batches. Close other applications to free up memory.
Issue: Print quality issues
Solution: Check printer settings and select high quality. Ensure paper is appropriate for ID cards (recommended: 200-250gsm cardstock). Adjust the DPI settings in your PDF reader when printing. Use "Export Images Only" and print through a photo editor for more control.
Issue: Application crashes on startup
Solution: Reinstall the application. Check for antivirus interference. Make sure your Windows is up to date. Check system requirements are met.

Error Recovery

If the application crashes while working:

  1. Restart IDLab
  2. Your pupil data should be auto-saved
  3. Use "Save Settings" from the File menu regularly to avoid losing configuration
  4. Check the "IDLab_feedback.txt" file in your Documents folder for error details

FAQ

How many students can I manage in IDLab?

IDLab can handle thousands of students, though performance is best with 500-1000 at a time. For larger schools, consider using the grade/form filter when generating cards.

What photo format works best?

JPG or PNG formats work well. Photos should be passport-sized (3x4cm or similar) with a neutral background. For best results, use the same dimensions for all photos.

Can I use IDLab for staff ID cards?

Yes, IDLab works for both student and staff ID cards. Just enter staff information in the pupil data section.

Does IDLab require internet connection?

No, IDLab works completely offline after installation. All features are available without internet connectivity.

Can I customize the card size?

The default card size is standard ID card size (85.6 × 54 mm). Custom sizes can be achieved by modifying templates.

How do I create my own templates?

You can design background templates using any image editor (like GIMP or Photoshop) and save them as PNG files. The recommended template size is 1050 × 680 pixels.

Can I backup my data?

Yes, use the "Export Pupil List" feature to save your data as CSV files. Keep these files as backups.

Is there a limit to how many profiles I can create?

No, you can create as many profiles as needed for different schools or businesses.
