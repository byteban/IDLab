Introduction

IDLab is a comprehensive solution for generating professional school ID cards, developed specifically for Zambian schools and businesses. Created by Emmanuel Ngwenyama, IDLab simplifies the process of creating identification cards for pupils, particularly for ECZ examination candidates, without requiring heavy software like Photoshop or CorelDRAW.

The application provides an end-to-end solution from data entry to final printing, with specialized features for managing photos, customizing card designs, and handling large batches of student IDs efficiently.

Features

Core Features

Import, export, add, edit, and search pupil information

Installation Guide

System Requirements

  • Windows 7 or higher
  • 4GB RAM (minimum)
  • 100MB free disk space
  • Internet connection (for activation and updates)

Installation Steps

  1. Download the installer: Visit IDLab Desktop App and save the installer file to your computer
  2. Run the installer: Locate the downloaded file (IDLab_Installer) and double-click to run, then follow the on-screen instructions
  3. First Launch: Open IDLab from the Start menu or desktop shortcut. The application will prompt you to set up a profile
  4. License Activation: Enter your license key when prompted. If you don't have one, you can request it through the application
  5. Templates (Recommended): Download the IDLab Templates and extract to an easily accessible location
  6. Mobile App (Optional): For photo capture, download the IDLab Android App and install on your Android device

Setting Up Your Profile

Create a Profile

  1. On first launch, select your user type (School or Business)
  2. Enter your school/business name, district, and address
  3. Click "Save Profile"

Switch Between Profiles

  1. Go to Settings → Switch Profile
  2. Select a profile from the list

Managing Pupil Data

Import Pupil Data

  1. Go to Pupils → Import Pupil List
  2. Select a CSV file with the required columns (Name, School, Grade/Form, Exam Number, Sex, D O B)

Add Pupils Manually

  1. In the "Pupil Data Entry" tab, click "Add Pupil"
  2. Fill in the required information
  3. Click "Save"

Edit Pupil Information

  1. Double-click on any cell in the data table
  2. Edit the information
  3. Press Enter to save

Search and Filter

  • Use the search box to find specific pupils
  • Use the grade/form dropdown to filter by level

Managing Photos

Set Up Photo Folder

  1. In the "Generate IDs" tab, click "Photos Folder"
  2. Select a folder containing pupil photos
  3. Photos should be named with exam numbers for automatic matching

Using the Renaming Wizard

  1. Go to the "ID Photo Tools" tab
  2. Click on the "Renaming Wizard" sub-tab
  3. Select an unmatched photo from the gallery
  4. Find and select the correct pupil from the list
  5. Click "Assign Photo to Pupil"

Checking Photo Status

  1. Go to the "Management" sub-tab
  2. Review which pupils have photos and which don't
  3. Use "Export Unmatched Lists" to get lists of pupils missing photos

Generating ID Cards

Set Up Output Location

Click "Output Folder" to select where cards will be saved

Customize Card Design

  1. Add school logo and coat of arms (optional)
  2. Choose between custom template or built-in template
  3. Set motto and year
  4. Customize colors using the color picker buttons

Preview and Generate

  1. Click "Refresh Preview" to see how the card will look
  2. Make adjustments as needed
  3. Select a category from the dropdown (Grade 7, Grade 9, etc., or "All")
  4. Click "Export to PDF" to create a multi-card PDF
  5. Or use "Export Images Only" from the Pupils menu to create individual image files

Back Side Information

  1. Enter text for the back of the cards in the "Back Info" field
  2. Click "Export Back Info to PDF" to create a separate PDF for printing on the reverse side

Printing

Preview Layout

Click "Print Layout Preview" to see how cards will be arranged on the page

Print from PDF

  1. Open the generated PDF in your preferred PDF viewer
  2. Print using your printer's duplex feature for double-sided cards

Troubleshooting

Common Issues

Issue: Photos not matching with pupil data
Solution: Ensure photos are named exactly with the exam number. Use the Renaming Wizard to manually match photos.
Issue: License activation fails
Solution: Check your internet connection. If the problem persists, contact support with your license key.
Issue: PDF export is slow or crashes
Solution: Reduce the number of cards by filtering to a specific grade. Try exporting in smaller batches.
Issue: Unable to see photo preview
Solution: Verify that photos are in JPG, JPEG, or PNG format. Ensure the photo folder path doesn't contain special characters.
Issue: Data not saving
Solution: Check that you have write permissions for the application folder. Use "Save Changes" button explicitly after making changes.

Error Recovery

If the application crashes:

  1. Restart IDLab
  2. Your data should be auto-saved
  3. Check the "IDLab_feedback.txt" file in your user folder for error details
  4. Submit feedback through Help → Submit Feedback

FAQ

How many pupils can I manage in IDLab?

IDLab can handle thousands of pupils, though for performance reasons we recommend working with 500-1000 at a time.

What photo format is recommended?

JPG or PNG formats work best. Photos should be passport-sized with good lighting and a neutral background.

Can I use IDLab for staff ID cards?

Yes, IDLab works for both student and staff ID cards. Just enter staff information in the pupil data section.

Is IDLab only for Zambian schools?

While designed with Zambian schools in mind, IDLab can be used by any educational institution or business.

How do I renew my license?

Go to Settings → License Management and enter your new license key.

Can I transfer my license to another computer?

Yes, licenses are tied to your account, not your computer. Log in with the same credentials on the new computer.

What should I do if I forget my password?

Contact support at ngwenyamae3118@gmail.com for password recovery assistance.
